## 工作站使用注意事项（必读）

本教程的另一个文件 "Workbench_Setup.md" 主要介绍了工作站环境的**配置方法**。在实际使用过程中，更重要的是**规范使用**，以避免破坏系统环境、影响他人工作，或给后续维护带来困难。以下注意事项请务必认真阅读并遵守。
(如果你尚不理解下文中的一些命令或名词，可先阅读“Wordbench_Setup.md"。)

---

### 1. 关于系统权限（重要）

1. **日常科研开发不需要使用 `sudo`**

   * 除非明确是在安装系统级软件（如 `apt install`），否则请避免使用 `sudo`。
   * 错误地使用 `sudo` 安装 Python 包，可能导致：

     * 系统 Python 被污染
     * Conda 环境失效
     * 其他用户环境异常

2. **禁止随意修改以下目录**

   * `/usr/bin`
   * `/usr/lib`
   * `/lib`
   * `/etc`
   * `/usr/local`（CUDA 除外，且需统一管理）

3. **任何不确定的系统级操作，请先询问**

   * 不清楚某条命令作用时，不要“试一下”。

---

### 2. 关于 Conda 虚拟环境的使用规范

1. **所有 Python 相关工作应该在 Conda 环境中进行**

   * 包括：

     * 深度学习代码
     * 强化学习环境
     * 数据分析脚本
   * 禁止在系统 Python（`/usr/bin/python`）中直接安装库。

2. **使用前确认当前环境**

   ```bash
   conda info --envs
   ```

   当前激活环境前会有 `*` 标记。

3. **不要在 base 环境中做科研开发**

   * base 环境仅用于 Conda 自身运行。
   * 建议每个项目使用独立环境，例如：

     ```bash
     conda create -n rl python=3.10
     conda activate rl
     ```

4. **不要随意删除他人 Conda 环境**

---

### 3. Conda / pip / apt 的正确使用方式

#### 3.1 conda install（首选）

适用场景：

* 科研计算
* 深度学习
* 数值计算

优点：

* 自动解决依赖冲突
* 与 CUDA、MKL 等库兼容性更好

示例：

```bash
conda install numpy scipy matplotlib pandas
```

---

#### 3.2 pip install（补充使用）

适用场景：

* Conda 中没有的库
* 一些最新研究代码依赖

注意事项：

* **必须在已激活的 Conda 环境中使用**
* 不要加 `sudo`

示例：

```bash
pip install gymnasium stable-baselines3
```

---

#### 3.3 apt install（原则上避免）

apt 安装的是**系统级软件包**，而非科研环境设计。

风险：

* Python 版本不可控
* 依赖固定，容易冲突
* 会影响系统中所有用户

允许使用的情况：

* 非 Python 工具（如 `htop`, `tmux`, `tree`）
* 编译工具（如 `build-essential`）

不推荐示例：

```bash
sudo apt install python3-numpy  # 不推荐
```
---
#### 3.4 推荐的安装优先级（重要）

在科研与实验环境中，**推荐始终遵循以下顺序**：

1. **优先使用 `conda install`**
2. Conda 无对应包时，再使用 `pip install`
3. **避免使用 `apt install` 安装 Python 库**

该顺序有助于：

* 降低依赖冲突风险
* 保持环境结构清晰
* 提高实验可复现性
---

### 4. 关于 Git 与代码管理

1. **代码一律放在用户主目录**

   * 推荐路径：

     ```text
     /home/username/projects/
     ```
   * 禁止在 `/usr`、`/opt` 等目录中写代码。

2. **公共机器慎用 SSH Key**

   * 优先使用 HTTPS + Token
   * 若使用 SSH：

     * 设置密钥密码
     * 使用完及时注销或清理 agent

3. **不要在公共仓库中提交**

   * 密钥文件（`.ssh`）
   * 数据集
   * 模型权重（除非明确需要）

---

### 5. 关于 VS Code 的使用习惯

1. **VS Code 只是编辑器，不是运行环境**

   * Python 解释器以 Conda 环境为准
   * 注意右下角 Python Interpreter 是否正确

2. **不要频繁安装不必要的扩展**

   * 扩展会影响启动速度和稳定性

3. **远程开发注意端口占用**

   * 关闭不用的 Jupyter Server
   * 避免长时间后台占用 GPU


---

### 6. 数据与文件管理

1. **不要把大数据放在主目录**

   * 建议使用统一的数据盘或指定目录

2. **重要数据及时备份**

   * 工作站不是长期存储设备

3. **清理无用文件**

   * 包括无用日志等

---

### 7. 遇到问题的基本原则

1. **不要“重装系统式解决问题”（重要）**

   * 随意删除 CUDA / Conda / 系统库是高风险行为
  
2. **先定位问题，再操作**

   * 看报错
   * 查当前环境
   * 确认 Python 路径


3. **无法自己解决报错时**

   * 保存和描述清楚：

     * 做了什么
     * 用了什么命令
     * 报错信息

---

## 总结

请规范使用工作站。
